## ❓ 常见问题

### Q: 推送时提示 "updates were rejected"

A: 需要先删除远程的 master 分支:
```bash
git push origin --delete master
git push -u origin main
```

### Q: 可以保留 master 分支吗?

A: 可以,但需要:
1. 在 GitHub 上设置 main 为默认分支
2. 或者修改项目使用 master

### Q: 删除 master 分支会影响已克隆的仓库吗?

A: 不会,只是影响新克隆的默认分支。已克隆的仓库需要手动拉取:
```bash
git fetch --all
git checkout main
```

### Q: Personal Access Token 在哪里获取?

A: 访问 https://github.com/settings/tokens,点击 "Generate new token (classic)"
