# 如何修改 GitHub 默认分支

## 两种解决方案

### 方案 1: 在 GitHub 上将 master 改为默认分支(推荐)

#### 步骤 1: 访问仓库设置
1. 访问: https://github.com/zmx199299/table-cleaner/settings/branches
2. 或者在仓库页面点击 "Settings"
3. 点击左侧的 "Branches"

#### 步骤 2: 修改默认分支
1. 找到 "Default branch" 部分
2. 点击 "🔘" 或编辑按钮
3. 选择 "master" 作为默认分支
4. 点击 "Update" 确认
5. 确认对话框(如果弹出)

#### 步骤 3: 保护 master 分支(可选但推荐)
1. 在同一页面,找到 "Branch protection rules"
2. 点击 "Add rule"
3. Branch name pattern: 输入 `master`
4. 勾选:
   - ✅ Require a pull request before merging
   - ✅ Require approvals: 1
   - ✅ Require status checks to pass before merging
5. 点击 "Create" 或 "Save"

### 方案 2: 修改本地和远程为 main 分支

#### 步骤 1: 重命名本地分支
```bash
cd '/mnt/c/Users/zmx19/OneDrive/私人/codelearn/表格自动化清理'
git branch -m master main
```

#### 步骤 2: 删除远程的 master 分支
```bash
git push origin --delete master
```

#### 步骤 3: 推送新的 main 分支
```bash
git push -u origin main
```

#### 步骤 4: 在 GitHub 上设置 main 为默认分支
1. 访问: https://github.com/zmx199299/table-cleaner/settings/branches
2. 在 "Default branch" 部分选择 "main"
3. 点击 "Update"

#### 步骤 5: 更新所有文件中的 master 为 main
需要更新以下文件:
- `.github/workflows/build.yml` 中的分支引用
- 文档中的分支说明

## 注意事项

1. **如果有 pull request**: 修改默认分支前,先合并或关闭所有 PR
2. **如果有其他分支**: 它们会基于新的默认分支重新计算关系
3. **CI/CD**: 确认 GitHub Actions 配置支持新分支
4. **文档**: 更新所有提到默认分支的文档

## 常见问题

Q: 修改默认分支会影响现有代码吗?
A: 不会,只是改变新克隆时默认使用的分支

Q: 可以随时修改默认分支吗?
A: 可以,在设置中随时可以切换

Q: 修改后,之前的 master 分支会删除吗?
A: 不会,除非你手动删除
